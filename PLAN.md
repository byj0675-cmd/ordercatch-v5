# OrderCatch 전체 구현 계획서

## 1. 오더캐치 주문서 웹훅
- [ ] `[오더캐치 주문서]` 없는 메시지 → 무시 (응답 없음)
- [ ] `[오더캐치 주문서]` 있는 메시지 → 해당 블록만 추출 → Gemini 파싱 → 장부 등록
- [ ] 자동 안내 메시지 제거
- [ ] Gemini 프롬프트: 자유 필드 전부 `options`에 저장하도록 명시

## 2. 날짜 이동 가능한 주문 뷰
- [ ] 상단 날짜 네비게이터 추가 (`←` 어제 / 오늘 / 내일 `→`)
- [ ] 날짜 클릭 시 날짜 선택기(input date) 팝업
- [ ] 선택된 날짜의 픽업 주문만 필터링 표시

## 3. 수기 주문 등록
- [ ] `+ 주문 등록` 버튼 추가
- [ ] 모달: 고객명 / 연락처 / 상품명 / 픽업일시 / 메모 / 상태 입력
- [ ] 기존 `/api/orders/register` API 재사용

## 4. 입금 처리 버튼
- [ ] 주문 카드에 `입금확인` 버튼 추가
- [ ] 클릭 시 상태 `입금대기` → `입금완료` 업데이트
- [ ] 즉시 UI 반영 (낙관적 업데이트)

## 5. 당일 주문 프린트
- [ ] `🖨️ 오늘 주문 출력` 버튼 추가
- [ ] A4 출력 레이아웃: 번호 / 고객명 / 상품 / 픽업시간 / 상태 표
- [ ] `@media print` CSS로 버튼·헤더 숨기고 표만 출력
